Сетевая архитектура "клиент-сервер" - это модель взаимодействия между компьютерами в сети, при которой 
один компьютер (сервер) предоставляет ресурсы или услуги, а другие компьютеры (клиенты) запрашивают 
эти ресурсы или услуги.

Клиентская часть приложения предназначена для взаимодействия с пользователем и отправки запросов на 
сервер для получения данных или выполнения определенных действий. Серверная часть приложения обрабатывает 
запросы от клиентов, предоставляет им запрошенные данные или выполняет необходимые операции.

Сетевой протокол - это набор правил и соглашений, определяющих формат и последовательность обмена 
данными между устройствами в компьютерной сети.

Стек протоколов - это иерархическая структура сетевых протоколов, организованных в виде нескольких 
уровней для обеспечения передачи данных между устройствами в сети. Обычно стек протоколов состоит из 
нескольких уровней, таких как физический, канальный, сетевой, транспортный, прикладной и другие.

В стек протоколов TCP/IP входят следующие протоколы: IP (Internet Protocol) - протокол маршрутизации 
и доставки данных, TCP (Transmission Control Protocol) - протокол управления передачей данных, UDP 
(User Datagram Protocol) - протокол без установления соединения, ICMP (Internet Control Message Protocol) - 
протокол управления сообщениями в сети и другие.

Сокет - это абстрактный конечный пункт взаимодействия между двумя устройствами в сети. Он представляет 
собой комбинацию IP-адреса и номера порта, которые идентифицируют конкретное сетевое соединение.

Номер порта - это числовой идентификатор, который определяет конкретное сетевое приложение или службу 
на устройстве. Порты используются для маршрутизации и доставки сетевых пакетов к нужному приложению или службе.

Для установки сетевого соединения необходимо создать сокет на каждом устройстве, затем установить связь 
между сокетами, отправляя и принимая данные через них.

Датаграмма - это формат данных, используемый в протоколе UDP (User Datagram Protocol). Он представляет 
собой блок данных, который передается между устройствами в сети без установления надежного соединения.

RMI (Remote Method Invocation) - это технология в Java, которая позволяет вызывать методы удаленных 
объектов через сеть. RMI обеспечивает механизм удаленного вызова методов, позволяя объектам в одной 
виртуальной машине Java вызывать методы объектов в другой виртуальной машине Java.
