Назначение и возможности JDBC?
JDBC (Java Database Connectivity) - это API для взаимодействия с базами данных из языка Java. 
Основное назначение JDBC - предоставить возможность Java-приложениям устанавливать соединение с 
базами данных, отправлять SQL-запросы и обрабатывать результаты.

Что такое JDBC драйвер? Какие бывают типы?
JDBC драйвер - это программное обеспечение, которое позволяет Java-приложениям взаимодействовать с 
базами данных через JDBC. Существует четыре типа JDBC драйверов:

JDBC-ODBC мост
Нативные API-драйверы
JDBC-сетевые протоколы
JDBC-технологии Java
Какие шаги (перечислите) нужно выполнить для извлечения данных?
Шаги для извлечения данных включают:

Установка соединения с базой данных.
Создание SQL-запроса.
Выполнение запроса с использованием Statement или PreparedStatement.
Обработка результатов с использованием ResultSet.
Как и где задать параметры соединения?
Параметры соединения, такие как URL базы данных, имя пользователя и пароль, обычно задаются в 
строке подключения JDBC в Java-коде или в конфигурационном файле.

Перечислите основные JDBC API компоненты.
Основные компоненты JDBC API включают:

DriverManager: управляет регистрацией и выбором JDBC-драйверов.
Connection: представляет соединение с базой данных.
Statement и PreparedStatement: используются для отправки SQL-запросов к базе данных.
ResultSet: представляет результат выполнения SQL-запроса.
DatabaseMetaData: предоставляет метаданные о базе данных.
Какая разница между execute, executeQuery, executeUpdate?

execute: используется для выполнения любых SQL-запросов.
executeQuery: используется для выполнения SELECT-запросов, которые возвращают набор результатов в виде ResultSet.
executeUpdate: используется для выполнения INSERT, UPDATE, DELETE и других DML-запросов, которые 
возвращают количество измененных строк.
Расскажите об интерфейсе ResultSet.
ResultSet - это интерфейс, который представляет набор результатов из базы данных. Он позволяет 
последовательно перемещаться по результатам и извлекать данные из них.

Как и для чего используют PreparedStatement?
PreparedStatement используется для выполнения предварительно скомпилированных SQL-запросов. Он 
обеспечивает более безопасное и эффективное выполнение запросов, так как позволяет использовать параметры для 
передачи данных в запрос.

Как и для чего используют транзакции?
Транзакции используются для обеспечения атомарности, согласованности, изоляции и долговечности операций 
с базой данных. Они позволяют группировать несколько операций в единый блок, который либо выполняется 
полностью, либо отменяется целиком.

Перечислите и поясните типы чтения данных в транзакциях. Что такое уровни изоляции?
Типы чтения данных в транзакциях:

READ_UNCOMMITTED: Позволяет читать незафиксированные данные другими транзакциями.
READ_COMMITTED: Позволяет читать только зафиксированные данные другими транзакциями.
REPEATABLE_READ: Гарантирует, что повторные чтения в рамках одной транзакции будут одинаковыми.
SERIALIZABLE: Обеспечивает максимальный уровень изоляции, предотвращая одновременное чтение и запись 
данных другими транзакциями.
Что такое DatabaseMetaData интерфейс?
DatabaseMetaData - это интерфейс, который предоставляет метаданные о базе данных, такие как список 
таблиц, индексы, хранимые процедуры и другие свойства базы данных.

Зачем используют шаблон DAO?
Шаблон DAO (Data Access Object) используется для разделения бизнес-логики приложения и доступа к 
данным. Он предоставляет абстрактный интерфейс для работы с данными, который скрывает детали реализации доступа к 
базе данных. Это позволяет легко изменять и заменять источники данных без изменения остального приложения.
